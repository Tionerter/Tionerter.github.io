# 目标导向型行动计划(GOAP)

目标导向型行动计划（GOAP）是Monolith的Jeff Orkin在游戏F.E.A.R.中首创的技术。从2005年开始，已经被用于许多游戏中，最近用于诸如Just Cause 2和Deus Ex：Human Revolution等游戏。 GOAP源自斯坦福研究院的AI问题解决器（STRIPS）方法，该方法最早于1970年代初开发。概括地说，STRIPS（和GOAP）允许AI系统自己创建解决问题的方法，通过提供游戏世界运作方式的描述-----即可能采取的行动的清单，每个行动可以被使用的需求（称为“前提条件”）和行动的效果。然后，AI系统用符号表示世界的初始状态以及需要实现的一些目标。在GOAP中，这些目标通常从希望NPC实现的预先设定的目标集合中选择的，这些目标是通过某种方法（例如优先级或状态转换）选择的。然后，计划系统可以确定一系列动作，以允许其控制的主体将世界从原始状态更改为包含满足其当前目标所需的事实的状态。在经典计划中，理想情况下，这将是达到目标状态的关键路径，而目标将是包含所有客观事实的最容易达到的状态。

GOAP的工作方式是“向后链式搜索”，它意味着从要实现的目标开始，确定要实现这些目标需要采取什么行动，然后继续确定要实现这些行动的前提条件需要做什么,以这种方式向后搜索，直到达到开始的状态。向后搜索是一种扎实的主力方法，尽管它不太优雅，但比起更现代的技术，它更易于理解和实施。

向后链式搜索的工作方式如下：

> ```
> •将目标添加到未完成的事实列表中
> •对于每个未完成的事实
>   •删除此未完成事实
>   •查找将此未完成事实作为结果的行动
>   •如果满足此行动的前提条件
> 	 --将行动添加到计划中
>      --向后搜索以将现在支持的动作链添加到计划中
>   •否则
>      --将此行动的前提条件添到未完成事实列表
> ```

GOAP最后一个有趣的方面是，它允许计划系统忽略“条件前提”，但必须在运行时满足条件才能执行操作。这样就可以推理绕开世界上某些无法轻易用符号表示的方面，例如确保在开火之前确保瞄准目标的视线，同时通过访问在计划期间不可用的信息来确保（确保搜索仍然易于处理） ），则可以满足这些约束。这使GOAP生成的计划具有一定的灵活性，并且它所要求采取的行动更多地应用于战术层面，而不是最基本的执行层面。也就是说，该计划告诉您该怎么做，但不一定告诉您如何去做。例如，省略了如何建立视线以开始拍摄的详细说明，并且可以更加被动地进行处理。

假设我们有一个经典的士兵NPC角色，其目标是杀死另一个角色。我们可以将此目标表示为Target.Dead。为了使目标死亡，角色需要向他射击（在基本系统中）。射击的先决条件是装备武器。假设我们的角色没有武器，那么我们现在需要一种可以给角色提供装备武器的行动，也许是从皮套上取下一个武器。当然，这需要其自身拥有武器作为前提-----即角色物品清单中有可用的武器。在这种情况下，我们刚刚创建了装备武器然后射击的简单计划。如果角色没有武器怎么办？然后，我们的搜索将不得不找到另一种方法。如果不可能，搜索可以回溯并寻找射击动作的替代方法。也许附近有一个安装好的武器可以用来提供Target.Dead效果，或者甚至是我们可以用车辆撞击。无论哪种情况，很明显，通过提供一组全面的对世界上可以做什么的动作选择，我们可以让角色决定应该做什么，让动态有趣的行为自然地出现，而不是让在开发过程中设想和创建它们。

最后，考虑一个武器拥有最大射程的游戏。作为上面游戏的前提，我们可以说目标必须在该范围内。计划系统不会花时间在搜索上试图做到这一点,这不可能，因为这涉及到目标可能如何运动的推论等,所以它不会发射武器，直到条件为真为止，或者改为使用其他战术，例如射程更远的其他武器。

基于自动计划的NPC控制方法有很多让人喜欢的地方。它通过允许设计师专注于创建将自动组装为行为的简单组件来简化开发过程，并且还可以提供“新颖”的解决方案，这可能是团队从未想过的，通常会为玩家提供出色的轶事复述。 GOAP本身仍然是自动化计划所能提供的最低限度的成果，并且从纯粹的科学观点来看，自从开发以来，GOAP的技术水平已经取得了显着进步。话虽如此，但如果正确使用它，它仍然可能是一种非常强大的技术，并且可以为特定的定制提供良好的，可适应的起点。

值得注意的是，这些采用以字符为中心的智力观的方法从开发团队中删除了很多创作和导演控制。能够“思考”自己的角色会在游戏世界中变成松散的大炮，制定的计划虽然有效地实现了角色的目标，但并未实现创造沉浸式和引人入胜的体验这一更广泛的目标，从而可能破坏电影场景例如，如果一个士兵的计划不能使他越过方便放置的“大红色桶”。

尽管可以通过使用知识工程技术和表示技巧来避免此类问题，但它并没有像行为树之类的架构那样简单，后者可以将所需的行为直接注入角色的决策逻辑中。同时，GOAP方法比基于分层任务网络的方法要容易得多，因为在GOAP中，您只需要描述世界中对象的机制即可。 

GOAP和类似技术不是灵丹妙药，但在正确的情况下，它们可以证明它们在创造真实行为和玩家可以建立密切关系的角色方面非常强大。
